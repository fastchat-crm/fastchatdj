Guia paso a paso — Conectar con Meta Cloud API
Requisito previo: la empresa debe estar verificada en Meta Business Manager (proceso que toma 1-3 semanas). Sin esto, Meta no habilita produccion. Ver requisitos de verificacion
  1. Paso 1 — Crea una App en Meta for Developers
    • Ir a developers.facebook.com/apps → My Apps → Create App → tipo Business.
    • Agregar el producto: WhatsApp Business Platform.
    • App ID: aparece debajo del nombre de tu App en la cabecera, y tambien en App Settings → Basic → primer campo. Son ~15 digitos.
    • App Secret: App Settings → Basic → campo App Secret → clic en Show (te pide tu password de Facebook) → copia los 32 caracteres alfanumericos. Sensible: no lo compartas.
    Copia en FastChat: App ID, App Secret
  2. Paso 2 — Registra un numero de WhatsApp
    • En tu App → WhatsApp → API Setup.
    • Veras un numero de testing gratuito (1.000 conversaciones/mes, ideal para probar).
    • Para produccion: clic en Add phone number y registra una linea dedicada.
    • Copia el WABA ID y el Phone Number ID.
    Copia en FastChat: WABA ID, Phone Number ID
  3. Paso 2b — Business Portfolio ID opcional
    • Ir a business.facebook.com/settings.
    • Mira la URL del navegador: …?business_id=XXXXXXXXXXXXXXX — ese numero largo es el Portfolio ID.
    • Alternativa: en el panel izquierdo Business Info → arriba del nombre del negocio aparece Business Manager ID.
    • No es lo mismo que el WABA ID. El Portfolio es el paraguas que contiene WABA, paginas FB, pixels, etc.
    • Lo necesitas solo si vas a operar con multiples WABAs bajo el mismo portfolio, o sincronizar plantillas a nivel portfolio.
    • Para un solo numero enviando mensajes: saltatelo.
    Copia en FastChat (si lo necesitas): Business Portfolio ID
  4. Paso 3 — Crea un System User y genera un Token permanente
    • Ir a business.facebook.com/latest/settings/system_users → Add.
    • Nombre sugerido: fastchatdj-integration, rol: Admin.
    • Asignale acceso a la App (Full Control) y a la WABA (Full Control).
    • Clic en Generate new token → elige la App → marca permisos: whatsapp_business_management, whatsapp_business_messaging, business_management.
    • Copia el token inmediatamente (empieza con EAAB...). Meta solo lo muestra una vez.
    Copia en FastChat: System User Access Token
    {# ── Aclaracion de tokens por canal ── #}
    Importante: cada canal Meta usa un token distinto.
    La misma App y App Secret sirven para los 3 canales, pero el token de acceso varia segun el canal:
    Canal Tipo de token Permisos / scopes
    WhatsApp Cloud System User token (este Paso 3) whatsapp_business_management
    whatsapp_business_messaging
    Instagram DM Page Access Token de la FB Page vinculada al IG Business instagram_basic
    instagram_manage_messages
    pages_messaging
    Facebook Messenger Page Access Token de la FB Page pages_messaging
    pages_show_list
    Si solo usas WhatsApp, basta con el System User token. Para activar IG o Messenger despues, vas a Business Settings → Pages, asignas la Page al System User y generas su Page Access Token.
    Lo que SI se comparte entre los 3: App ID, App Secret, y la misma URL de webhook (Meta envia eventos diferenciados por object: whatsapp_business_account, instagram, page).
  5. Paso 4 — Guarda la configuracion en FastChat
    • Pega los valores obligatorios: WABA ID, Phone Number ID, Access Token.
    • Clic en Guardar config Meta.
    • FastChat generara automaticamente un Verify Token.
  6. Paso 5 — Configura el Webhook en Meta
    • developers.facebook.com → tu App → WhatsApp → Configuration → Webhook → Edit.
    • Callback URL: la URL que FastChat muestra arriba.
    • Verify Token: el generado en el paso 4.
    • Clic Verify and save. Meta debe responder "Verified ✓".
    • Suscribete a: messages y message_template_status_update.
  7. Paso 6 — Verifica la conexion end-to-end
    • Clic en Verificar conexion con Meta.
    • Si todo esta OK, se rellena el numero, quality rating, y tier.
    • La sesion pasa a estado Conectado.
  8. Paso 7 — Crea plantillas (mensajes fuera de 24h)
    • Ir a /whatsapp/plantillas/.
    • Nueva plantilla → nombre, idioma, categoria, cuerpo con {% verbatim %}{{1}}, {{2}}{% endverbatim %}.
    • Guardar y someter a Meta → FastChat actualiza el estado automaticamente.
Consejo: Si te trabas, revisa la documentacion oficial de Meta o consulta /whatsapp/trazas/ para diagnosticar.

Como conectar Instagram DM
Requisitos previos: tener una cuenta Instagram Business (no personal) + una Pagina de Facebook linkeada a esa cuenta IG + la misma App de Meta for Developers que usas para WhatsApp. Docs oficiales
  1. 1. Convertir tu cuenta IG a Business (si no lo es).
    App Instagram → Configuracion → Cuenta → Cambiar a cuenta profesional → Empresa.
  2. 2. Linkear tu cuenta IG con una FB Page.
    facebook.com/tu-page → Configuracion → Instagram → Conectar cuenta. Si no tenes Page, creala primero en facebook.com/pages/create.
  3. 3. Habilitar "Acceso a mensajes" en tu cuenta IG.
    App Instagram → Configuracion → Privacidad → Mensajes → Permitir acceso.
  4. 4. Agregar el producto Instagram a tu Meta App.
    developers.facebook.com/apps → tu App → Add Product → seleccionar Instagram + Messenger.
  5. 5. Generar el Page Access Token.
    Business Settings → Pages → tu Page → Add People / System User → click Generate Token.
    Permisos requeridos: instagram_basic, instagram_manage_messages, pages_messaging, pages_show_list.
    Es un token de Page, NO el System User token de WhatsApp.
  6. 6. Obtener el IG User ID (IGSID).
    Llama Graph API: GET /me/accounts?access_token={page_token} → en cada Page veras instagram_business_account.id (17 digitos). Ese es el IGSID.
  7. 7. Configurar el webhook IG en Meta.
    App → Webhooks → Instagram → URL: la que se autogenera al guardar abajo. Verify Token: el campo Verify Token de este formulario. Suscribirse a: messages, messaging_postbacks.
  8. 8. App Review (para uso fuera de tu cuenta).
    Meta exige App Review para que tu integracion mande DMs a usuarios reales. Mientras estas en modo development, solo respondes a admins/testers.
Tabla de IDs Instagram:
  • IG User ID (IGSID): 17XXXXXXXXXXXXXX — el ID de tu cuenta IG Business.
  • Page ID: el ID de la FB Page linkeada (mismo que usa Messenger).
  • Page Access Token: token de la Page (NO User token, NO System User token de WA).

Primera vez configurando Messenger?
Revisa la guia paso a paso para saber como generar el Page Access Token y configurar el webhook.
Como conectar Facebook Messenger
Requisitos previos: tener una Pagina de Facebook + la misma App de Meta for Developers que usas para WhatsApp/IG + ser admin de la Page. Docs oficiales
  1. 1. Tener una FB Page como admin.
    facebook.com/pages/create si no tenes una. Para integracion empresarial usa la misma Page que IG.
  2. 2. Agregar el producto Messenger a tu Meta App.
    developers.facebook.com/apps → tu App → Add Product → Messenger.
  3. 3. Conectar tu Page a la App.
    App → Messenger → Settings → Add or remove Pages → seleccionar tu Page.
  4. 4. Generar el Page Access Token.
    App → Messenger → Settings → seccion Access Tokens → click Generate Token al lado de tu Page (acepta los permisos en el popup).
    Permisos requeridos: pages_messaging, pages_show_list.
    Para uso avanzado: pages_manage_metadata (para configurar greeting, persistent menu).
  5. 5. Obtener el Page ID.
    facebook.com/tu-page → Acerca de → al final de la pagina.
    O via Graph: GET /me?access_token={page_token} → campo id.
  6. 6. Configurar webhook Messenger.
    App → Messenger → Settings → Webhooks → Add Callback URL: la que se autogenera al guardar abajo. Verify Token: el campo Verify Token de este formulario. Suscribirse a: messages, messaging_postbacks, message_deliveries.
  7. 7. Suscribir tu Page al webhook.
    Toggle ON al lado de tu Page en la misma seccion. Sin esto, los webhooks no llegan.
  8. 8. App Review (para usuarios fuera de admins/testers).
    Meta exige App Review para mandar DMs a usuarios externos. Mientras esta en development, solo Page admins/testers pueden conversar.
Resumen de tokens FB:
  • User Access Token: temporal, sirve para login. NO usar en backend.
  • Page Access Token: permanente (si lo generas con System User), bound a la Page. Es el que va en este form.
  • App Access Token: para llamadas server-to-server de la App. NO sirve para Messenger.
Message Tags (fuera de 24h): Para iniciar conversaciones o responder despues de 24h debes incluir un messaging_tag en el envio. Tipos validos:
  • CONFIRMED_EVENT_UPDATE — recordatorios de eventos confirmados.
  • POST_PURCHASE_UPDATE — actualizaciones post-compra (envio, factura).
  • ACCOUNT_UPDATE — cambios en cuenta del usuario (login, perfil).
  • HUMAN_AGENT — escalado a agente humano (requiere permiso especial).
