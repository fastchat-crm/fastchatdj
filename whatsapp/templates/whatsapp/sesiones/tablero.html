WABA ID y Phone Number ID son IDs diferentes — ambos numéricos largos (~16 dígitos). NUNCA uses el número telefónico (ej. 0959843712) como Phone Number ID. El Phone Number ID es un ID interno de Meta que vive dentro de la WABA.
  1. 1
    WABA ID ~16 dígitos

    Abrí WhatsApp Manager. Cada cuenta WABA muestra su ID en la cabecera (ícono al lado del nombre). Alternativa: Business Settings → Cuentas → WhatsApp Accounts → seleccionar la WABA → ID arriba.

    Abrir WhatsApp Manager Ver en Business Settings
  2. 2
    Phone Number ID ~16 dígitos · ID interno

    Dentro del WhatsApp Manager → seleccioná tu WABA → pestaña Phone Numbers. Cada número tiene un ícono al lado para copiar su Phone Number ID. Si abrís el detalle del número, también aparece arriba como Phone number ID.

    Abrir números de WhatsApp WhatsApp Manager
    Si pegás el número telefónico (0959843712), Meta responde "Object does not exist". Es otro ID.
  3. 3
    Número visible opcional

    El número telefónico real, formateado para mostrar (ej. +593 95 984 3712 o 0959843712). Si lo dejás vacío, lo trae Meta al validar.

  4. 4
    Business Account ID opcional ~15 dígitos

    En Business Settings → Información del negocio. Arriba aparece el Business ID debajo del nombre del Business. {% if bid %}
    Ya lo tenés cargado en CredencialMetaApp: {{ bid }} — pegá ese mismo valor. {% endif %}

    Abrir Business Info
  5. 5
    Access Token (System User long-lived) empieza con EAAG o EAAB
    1. Abrí Business Settings → Users → System Users y seleccioná uno tipo Admin (creálo si no existe). Abrir System Users
    2. Click Add Assets → categoría WhatsApp Accounts → seleccioná la WABA → marcá Full control → Save Changes.
    3. Click Generate New Token → seleccioná tu Meta App → marcá los scopes:
      • whatsapp_business_management
      • whatsapp_business_messaging
      • business_management
    4. Elegí Never expires → Generate Token → copiá el token una sola vez (Meta no lo vuelve a mostrar) → pegalo en el campo de abajo.
    Lo guardamos cifrado con Fernet. Nunca lo pegues en Slack, mail ni issues.
Antes de guardar, pulsá Validar con Meta: probamos los IDs contra Graph API y autorrellenamos lo que podamos detectar.
