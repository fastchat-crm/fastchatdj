Endpoint operativo

Este endpoint es para Meta, no para navegadores

Meta llama esta URL en dos situaciones:

  1. Verificación GET
    Una sola vez al configurar el webhook en Meta Developer Portal. Meta envía ?hub.mode=subscribe&hub.verify_token=...&hub.challenge=... y esperamos el challenge en texto plano.
  2. Eventos en tiempo real POST
    Cada mensaje entrante, status (sent/delivered/read), cambio de plantilla, etc. Body JSON firmado con HMAC-SHA256 en X-Hub-Signature-256 usando el app_secret.

💡 Si llegaste aquí desde un navegador verás este panel informativo. Meta no lo verá — para Meta respondemos texto plano con el challenge o un código HTTP apropiado.
