{# ═══════════════════════════════════════════════════════════════════ #} {# PASO 1 — Identidad #} {# ═══════════════════════════════════════════════════════════════════ #}
¿Qué es un departamento?
Es una sección del chatbot con un propósito claro. Ejemplo: Matrículas Soporte técnico Información general Becas
Más adelante (paso 3) definirás el flujo que se ejecuta dentro de este depto.
{{ form.nombre }}
Nombre corto y claro. Se verá en los menús del bot.
{{ form.color }}
Solo para distinguirlo visualmente en el panel.
{{ form.mensaje_saludo }}
Se envía al cliente al entrar al depto. Puedes usar variables: {% verbatim %} {{contacto.nombre}} {{contacto.numero}} {{sesion.nombre}}
Ejemplo: 🎓 Hola {{contacto.nombre}}, bienvenido al área de Matrículas. ¿En qué te ayudo? {% endverbatim %}
{# ═══════════════════════════════════════════════════════════════════ #} {# PASO 2 — Enrutamiento #} {# ═══════════════════════════════════════════════════════════════════ #}
¿Cómo llegan los mensajes a este departamento?
El bot decide el depto correcto en este orden:
1️⃣ Coincidencia de palabras clave en el mensaje.
2️⃣ Número de menú (cuando el cliente elige "1", "2", "3"…).
3️⃣ Departamento de entrada de la sesión (configurado en Sesiones).
4️⃣ Departamento marcado como por defecto.
{{ form.palabras_clave }}
Una palabra o frase por línea. Si el mensaje del cliente contiene alguna, se enruta a este depto sin mostrar menú.
Ejemplo para "Matrículas": matricula matrícula inscripción pago saldo
{{ form.es_default }}
Si ningún otro depto matchea, el bot cae aquí. Solo un depto debería estar marcado como default por sesión.
{{ form.activo_tradicional }}
Si lo apagas, el depto NO responde con flujo tradicional — queda solo para handoff/asignación humana.
{# ═══════════════════════════════════════════════════════════════════ #} {# PASO 3 — Flujo de opciones (árbol) #} {# ═══════════════════════════════════════════════════════════════════ #}
Construye el flujo que verá el cliente.
Cada opción es un nodo. Elige su tipo según lo que haga:
MENÚ lista opciones al usuario   RESPUESTA envía un texto fijo   PREGUNTA captura un dato   HTTP llama a una API   CONDICIONAL if/else   HANDOFF pasa a humano   FIN cierra flujo
💡 Los tipos avanzados (HTTP, condicional, switch) requieren configuración JSON que se edita en Django Admin hasta que esté el editor visual.
    📖 Ejemplo — Centro de atención estudiantil:
    • MENÚ "¿En qué te ayudo?" → hijos: "Matrículas", "Notas", "Becas"
    • PREGUNTA "¿Cuál es tu cédula?" (validación: cedula)
    • HTTP GET a /alumnos/{cedula} → guarda nombre/saldo
    • RESPUESTA {% verbatim %}"Hola {{variables.nombre}}, tu saldo es {{variables.saldo}}"{% endverbatim %}
    • FIN cierra
